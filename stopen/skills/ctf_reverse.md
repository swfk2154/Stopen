# CTF 逆向 — Reverse 题目

## 目标
分析二进制文件，找出隐藏的 flag。

## 方法

### 1. 文件分析
- 识别文件类型（ELF/PE/MachO）
- 查看可读字符串：`strings`
- 检查文件头信息

### 2. 静态分析
- 查找硬编码的字符串（flag、password、key）
- 分析关键函数逻辑
- 对比输入输出的变换关系

### 3. 动态分析
- 设置断点在关键比较处
- 跟踪数据流：输入 → 变换 → 比较
- 如果 flag 是逐字符比较：逐位爆破

### 4. 常见算法识别
- XOR 加密：用已知明文反推密钥
- 简单替换：建立映射表
- 自定义 Base64：替换了码表的 Base64

### 5. 破解思路
- 如果程序比较 flag：修改跳转条件
- 如果程序解密 flag：修改密钥参数
- 如果程序算法复杂：跟踪关键内存的最终值

## 提示
- 先用 `crypto` 工具尝试常见编码和简单 XOR
- flag 通常以明文形式存在于二进制文件的某个位置
