# 报告生成 (Report)

## 目标
将渗透测试过程和结果整理成规范的报告。

## 报告要素

### 1. 基本信息
- 任务名称、目标地址、测试时间
- 测试人员、工具版本

### 2. 信息收集结果
- 开放端口列表（带服务版本）
- Web 技术栈
- 子域名列表
- 发现的敏感路径

### 3. 漏洞列表
- 每个漏洞：名称、严重程度、位置、复现步骤
- 证据截图、请求/响应原文
- CVE 编号（如适用）

### 4. 利用过程
- 成功利用的漏洞：利用方法、工具参数
- 获取的权限级别
- C2 会话信息（IP、用户、OS）

### 5. 风险与建议
- 修复建议（漏洞对应的修复方案）
- 风险等级评估（高危/中危/低危）

### 6. CTF 报告
- 各题目的 flag 值和获取方法
- 使用的技能和工具
- 解题思路和耗时

## 输出格式
- Markdown 报告（存储到 storage/reports/）
- 包含 Python PoC 代码（如适用）
